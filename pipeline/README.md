# 데이터 파이프라인

빌드타임에 실행되어 시군구별 정적 JSON과 bounding-box 인덱스를 만든다.
**주 1회 재실행**된다(8월 운영 루틴) — 멱등하게 설계되어 있다.

## 실행

```bash
pip install -r requirements.txt

# 목 모드 — 키 없이. 앱 개발·데모용.
python build_data.py --mock

# 실데이터 — DATA_GO_KR_SERVICE_KEY 필요
export DATA_GO_KR_SERVICE_KEY=...
python build_data.py --city-codes 23,25
```

## 산출물

| 경로 | 용도 | 배포 |
|---|---|---|
| `app/src/data/sgg-index.json` | 좌표 → 시군구 매핑 bbox 테이블 | **앱 번들에 포함** (크기 민감) |
| `app/public/data/stops/{code}.json` | 시군구별 정류장 + 사전결합 쉼터 | CDN |

## 원칙

- **결측을 추정으로 메우지 않는다.** 운영시간이 없으면 `hours: null` → 앱이 "운영시간 미확인"을 표기한다.
  `09:00-18:00` 같은 그럴듯한 기본값을 채우는 순간 앱이 거짓말을 한다.
- **조인 실패한 정류장을 버리지 않는다.** `arrivalSupported: false`로 남긴다.
  버리면 지도에서 사라지고, 사용자는 "정류장이 없다"고 오해한다. 남기면 "도착정보 미제공"을 정직하게 말할 수 있다.
- **모든 JSON에 `dataDate`를 박는다.** 없으면 오래된 쉼터가 "있음(확인)"으로 표시된다 — 3상태 원칙 위반.
- **`shelterSearched` 플래그**: 빈 쉼터 배열이 "없음(확인)"인지 "미확인"인지를 가르는 유일한 정보다.

## 실데이터 전환 전 확인 필요 (사람)

1. **무더위쉼터 원천**: 행안부 API vs CSV 스냅샷. **운영시간 필드 포함 여부**가 관건 — 없으면 전국 쉼터가 전부 "운영시간 미확인"이 된다.
2. **정류장 ID ↔ TAGO nodeId 조인**: 샘플 지역 2~3곳(서울·광역시·중소도시)에서 위치 DB의 ID로 도착정보 API가 응답하는지 확인. 불일치면 매핑 테이블을 여기서 생성한다.
3. **배차간격 확보율**: 리포트의 이 숫자가 낮으면 **대안 정류장 제안이 화면에 거의 뜨지 않는다**(4중 조건 #4). 실측 후 7/21 컷 판단의 근거로 쓴다.
