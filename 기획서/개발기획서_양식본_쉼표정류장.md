## 2. 추진배경

**ㅇ 춘천 시민 민원 1위는 교통이고, 정류장 시설 민원은 계절을 가리지 않는다.**
- 팀이 춘천시 '시민의 소리' 최근 민원 1,000건(2025.05~2026.06)을 직접 크롤링하여 전수 분석한 결과, 교통·도로 관련 민원이 199건(19.9%)으로 전 분야 중 가장 많았고, 그중 버스·정류장·노선 관련이 74건이었다.
- 이 가운데 **버스정류장의 시설·대기환경을 직접 지목한 민원이 최소 12건, 1월부터 12월까지 연중 분포**한다. 실제 민원 제목: "버스 정류장 의자 상태 불량"(2026.05), "온의동 시내버스 승강장 버스정보표시기 설치"(2025.10), "스마트 버스정류장 추가 설치 요청"(2025.08), "우두동 노인복지관 앞 버스정거장 시설"(2026.01).
- 특히 **"남춘천역 버스정류장 의자 난방 중지 요청(2차)"(2025.06)** — 6월까지 온열의자 난방이 켜져 있었다는 이 민원은, 정류장 대기환경이 계절 단위로 관리되지 못하고 있음을 보여주는 실증이다.

**ㅇ 왜 노선·배차가 아니라 '정류장 대기환경'인가.**
- 같은 74건에서 가장 큰 단일 유형은 노선·배차·운행시간 불편(23건)이다. 그럼에도 정류장 대기환경을 택한 이유는 세 가지다. ① 노선·배차 조정의 핵심 근거(기점-종점 통행량 OD, 운행 비용, 운수사 계약)는 비공개이거나 폐쇄 분석 환경 전용이라, 공개 데이터만으로는 검증 가능한 서비스가 아니라 제안서에 그친다. ② 춘천시는 2025년 11월 노선 조정을 이미 시행했다 — 노선 불편은 민원으로 접수되고 행정이 대응 중인 문제인 반면, 정류장 대기환경은 민원 채널이 포착하지 못하는 사각지대다(아래 참조). ③ 정류장 시설 정보는 공개 데이터 결합만으로 당장 시민이 쓰는 서비스와 행정이 쓰는 설치 우선순위 근거가 되고, 그늘막·벤치 설치와 온열의자 계절 전환은 시가 저비용으로 즉시 집행할 수 있는 영역이다.

**ㅇ 폭염 민원은 0건이었다 — 그리고 그것이 문제의 본질이다.**
- 같은 1,000건에서 정류장의 더위·폭염을 직접 지목한 민원은 0건이었다. 이는 문제가 없다는 뜻이 아니다. 민원은 시민이 '행정이 고칠 수 있다고 믿는 형태'로만 접수된다 — 시민은 "덥다"고 민원을 넣지 않고 "의자를 고쳐 달라", "스마트 정류장을 설치해 달라"고 넣는다. 더위·추위 노출은 민원 채널이 포착하지 못하고 온열질환 통계로만 사후에 나타나는, 사전 예방이 필요한 영역이다.
- 배차간격이 긴 중소도시 특성상 정류장 대기 시간은 짧지 않다("버스 배차간격이 엉망입니다" 2026.02 등 배차 민원 다수). 그 대기가 한여름 뙤약볕, 한겨울 칼바람 아래에서 이루어질 때, 그늘막·벤치·쉘터 없는 정류장은 고령자 등 취약계층에게 건강 위험 지점이 된다.
- 팀이 승하차 원본 데이터(2025.6월 말 표본, 승차 26.8만 건)를 직접 분석한 결과, 한낮(12~15시) 승하차 상위 정류장에 중앙로입구·명동입구·남춘천역과 함께 **인성병원 정류장(4위, 동일 명칭의 양방향 정류장 합산 기준)**이 올라 있다 — 하루 중 가장 더운 시간대에 병원 앞 정류장 이용이 최상위권이라는 실측은, 폭염기 대기환경 문제가 의료 이용 취약계층과 겹치는 지점임을 보여준다.

**ㅇ 춘천시는 대응 인프라를 이미 갖고 있으나, '어디에 먼저'라는 근거가 공개 데이터로 확인되지 않는다.**
- 춘천시는 폭염대비 접이식 그늘막, 공공 벤치, BIT, 온열의자(민원으로 존재 확인)를 운영·관리하고 있다. 설치·관리 사업은 이미 존재한다.
- 특히 춘천시는 **유개식 승강장(쉘터) 비율을 50%(716곳)에서 70%(980곳)로 확대하는 사업을 2026년까지 추진 중**이다(언론 보도, 2024.7). 즉 "**앞으로 약 264곳의 쉘터를 어느 정류장에 세울 것인가**"라는 질문에 이미 예산과 목표가 배정되어 있다 — 본 서비스의 우선순위 대시보드는 정확히 이 질문에 답하는 도구다.
- 그러나 이 시설들이 실제 이용량이 많은 정류장, 취약계층이 많이 이용하는 정류장에 배치되어 있는지를 검증한 자료는 공개 데이터에서 확인되지 않는다. 수요 데이터와 시설 데이터가 각각 따로 존재할 뿐이다.

**ㅇ 해결하고자 하는 문제(문제정의)**
- ① 시민 관점: 내가 기다릴 정류장에 무엇이 있는지(의자·도착정보·그늘·조명)를 미리 알 방법이 없다 — 사계절 내내.
- ② 행정 관점: 그늘막·벤치·BIT 설치와 온열의자 계절 전환 관리를 결정할 때 쓸 수 있는, 이용량·시설·취약계층·민원을 정류장 단위로 결합한 근거 데이터가 없다.
- 본 과제는 이 두 문제를 하나의 데이터 구조로 해결하는 **사계절 플랫폼**이며, 본선 시연(7.31~8.1)은 발표 당일 시민 체감이 가장 큰 **폭염 레이어(v1)**로 진행한다.

## 3. 개발목표 및 내용

**ㅇ 개발 목표**
- 춘천시 공공데이터와 자체 구축 데이터만으로, 무박 2일 내 시연 가능한 웹 플랫폼(모바일 우선 반응형)을 완성한다.
- 목표 산출물: ① 시민용 '쉼표 정류장' 지도 서비스 ② 행정용 설치 우선순위 대시보드 ③ 전 과정 재현 가능한 데이터 처리 파이프라인(공개 저장소).

**ㅇ 서비스 구성 (2개 화면군)**

**[시민용] 쉼표 정류장 — "오늘, 어디서 기다릴까"**
- 현재 위치 또는 정류장명 검색 → 해당 정류장의 **대기환경 카드**: 그늘(인근 그늘막) / 벤치 / BIT(도착정보 안내기) / 야간조명(반경 30m 내 가로등) 4개 항목 + 혼잡 시간대 + 경유 노선.
- **3상태 표기 원칙**: 모든 시설 항목은 [있음(확인)] / [없음(확인)] / [미확인]의 3단계로만 표시한다. 좌표 근접 매칭만으로는 '없음'을 단정하지 않으며, 근접 데이터 부재 시 "반경 30m 내 데이터 없음 — 현장 미확인"으로 표기한다. 모든 카드에 데이터 기준일과 확인 방법을 명시한다.
- **인근 시설 우위 정류장 안내 (출발 전 대기 계획 도구)**: 4중 조건을 모두 만족할 때만 제안 — ① 동일 노선이 정차하는 인접 정류장 ② 직선거리 300m 이내(도보 시간은 분속 60m 고령자 보수 기준으로 환산, "직선거리 기준" 병기) ③ 확인된 시설 수가 명확히 우위(미확인 항목은 판정에 사용하지 않음) ④ 도보 예상시간 ≤ 해당 노선 배차간격의 1/2(배차간격 확인 불가 노선은 제안 비활성화). 실시간 도착 추격용이 아닌 출발 전 계획 도구이며, BIT가 있는 대안 정류장을 우선 정렬한다.
- 계절 모드 배너(폭염특보/한파/평시 — v1 시연은 폭염 모드), 인근 공원(반경 300m, 그늘 대체 쉼 공간) 표시.
- **"이 정류장 정보가 다른가요?" 시민 제보 버튼**: 데이터 공백을 시민 참여로 채우는 성장형 데이터셋 구조(v1은 접수 화면까지 구현).

**[행정용] 설치 우선순위 대시보드 — "다음 그늘막은 어디에"**
- **계절 프리셋 4종 + 조작 가능 슬라이더**:
  - 여름: 한낮(12~16시) 승하차 상위 25% AND 그늘막·쉘터 확인 안 됨 → 그늘막 설치 후보
  - 겨울: 출근(06~09시) 승하차 상위 AND 쉘터 확인 안 됨 → 바람막이·온열의자 후보
  - 야간: 심야 하차 상위 AND 반경 30m 내 가로등 0 → 조명 보강 후보
  - BIT: 이용량 상위 AND BIT 없음 → BIT 설치 후보 (관련 실민원 첨부 가능)
- 후보별 **근거 카드**: 시간대별 승하차 실측치, 소속 행정동의 독거노인 수(보조 지표), 인근 기존 시설 거리, 관련 민원 제목 원문. 후보 리스트는 표(CSV)로 즉시 내보내기 가능.
- 슬라이더(시간대·상위 비율·반경·취약지표 on/off)를 심사위원·공무원이 직접 조작 가능 — 임의 가중치 산식이 아니라 설명 가능한 조건임을 시연으로 증명.
- 그늘막·쉘터 설치는 보도폭·도로점용·지하매설물 등 현장 제약이 최종 결정을 좌우함을 인지하며, 본 대시보드는 **후보 발굴 단계의 근거 자료를 제공하는 도구**로 포지셔닝한다.

## 4. 주요특징 및 핵심기술

**ㅇ 컨셉 — "기다림을 쉼표로": 정류장 대기 경험이라는 미개척 문제**
- 버스 서비스 개선(노선·배차·도착정보)이 아니라 **정류장에서 기다리는 경험의 질**을 서비스 대상으로 정의한다. 교통 데이터와 생활안전 인프라 데이터(그늘막·벤치·가로등)를 정류장 단위로 결합한 사례는 춘천에서는 공개적으로 확인되지 않는다.
- 문제는 사계절(시설 민원 연중 12건+)이고, v1 시연은 본선 시점(7.31~8.1) 체감 전달력이 최대인 폭염 레이어를 선택했다.

**ㅇ 핵심기술 1 — 4축 데이터 결합분석과 실패 내성 Tier 설계**
- 수요 × 환경 × 시민 목소리 × 취약계층을 정류장 단위로 통합하되, 결합 키가 서로 다른 데이터(정류장번호·명칭·주소·행정동·좌표)를 표준화 테이블과 좌표 근접 매칭으로 잇는다. 조인 실패 시나리오 8종을 사전 식별하고 대체 경로를 설계해, 어떤 외부 데이터가 예상과 달라도 시연이 성립한다. 전 과정을 재현 가능한 코드로 공개한다.

**ㅇ 핵심기술 2 — 임의 가중치 없는 '설명 가능한 조건 필터' + 3상태 정직 표기**
- "위험지수 78점" 같은 합성 점수를 만들지 않는다. 심사위원과 공무원이 즉시 검증할 수 있는 조건식만 쓰고, 조건은 화면에서 직접 조작할 수 있다.
- 시설 정보는 [있음(확인)/없음(확인)/미확인] 3상태로만 표기한다. 데이터가 말할 수 없는 것을 단정하지 않는 것 — 이것이 공공 서비스 신뢰의 최소 조건이라고 본다.

**ㅇ 핵심기술 3 — 자체 구축 데이터 2종**
- 공개 데이터의 공백을 팀이 직접 메웠다: ① 민원 1,000건 크롤링·분류 데이터셋 ② 주요 정류장 150곳 승차대·의자 로드뷰 실사 데이터셋(경유 노선 수 상위 기준 선정, '현장 확인' 배지의 원천). 시민 제보 기능이 이 데이터셋을 성장시킨다.

**ㅇ 유사 서비스 대비 차별성**
- 카카오맵·네이버지도·버스 도착 앱: 도착 시간은 알려주지만 **기다리는 환경**(그늘·의자·조명 유무)은 국내 주요 지도·버스 앱에서 확인되지 않는다(2026.7 기준).
- 타 지자체 폭염 지도(무더위쉼터 안내): 시설 위치 나열에 그치고 이용량·취약계층 데이터와 결합되지 않으며, 계절이 지나면 효용이 사라진다. 본 서비스는 이용량이 실측되는 대기 지점(정류장)에 사계절 환경 데이터를 결합하고, 시민 안내와 행정 우선순위를 같은 데이터 구조에서 동시에 산출한다.

**ㅇ 정직한 한계 명시**
- 정류장 승차대(쉘터)·의자의 전수 데이터는 공개되어 있지 않다 → 자체 실사 150곳 + 3상태 표기 + 시민 제보 + 데이터 공개 요청(6절)의 4중 대응. 검증되지 않은 정류장에 '없음'을 표시하지 않는다.
- 버스민원 설문은 2022년 구축 자료이고 조인 키가 불확실하다 → 확정 기능이 아니라 발표 근거 통계로만 사용, 최신 경향은 시민의 소리 민원(2025~2026)으로 보완.
- 그늘막 데이터는 주소 기반(좌표 없음)이며 다수가 횡단보도 설치 시설이다 → 지오코딩 후 "인근 그늘막(도보 O분)"으로만 표기.
- 취약계층 통계는 행정동 단위라 정류장 단위 결합에 생태학적 한계가 있다 → 보조 지표로만 사용함을 화면에 명시.
- 승하차 데이터는 원본 검증 결과 공개 파일이 65,535행(구형 엑셀 행 한계)에서 절단되어 2025.6.25~28 약 4일 표본만 담고 있고, 노선번호 32개가 엑셀 날짜 자동변환으로 오염되어 있으며, 하차는 미태그로 과소집계된다 → 4일 표본 기준임을 화면에 명시, 노선번호는 노선아이디 기준 복원, 주 지표는 승차 기준. 전체 기간 원본은 6절에서 공식 요청.
- 승하차 표본(2025.6)은 2025.11.28 노선 조정 이전 수집분이다 → 노선 단위 해석은 하지 않고 **정류장 단위 수요 순위**로만 사용한다.

## 5. 기대효과 및 활용방안

**ㅇ 시민 체감 효과**
- 계절·시간대에 맞춰 대기환경이 나은 정류장을 미리 선택 — 고령자, 영유아 동반 보호자, 학생에게 즉각적 효용. "정류장에 무엇이 없는지"의 공개 자체가 시민 알 권리 충족이며 개선 요구의 근거가 된다.

**ㅇ 행정 활용 효과 (정책 연계)**
- 춘천시가 이미 매년 집행하는 그늘막·벤치·BIT 설치 예산과 온열의자 계절 전환 관리에, 이용량·취약계층·민원 실측 기반 후보 목록을 제공 — 신규 예산 없이 기존 사업의 효율을 높이는 즉시 적용형 제안.
- **본선 당일 '성과보고 한 장'이 나온다**: 시연과 동시에 "한낮 이용량 상위 + 그늘 미확인 정류장 설치 후보 TOP 10" 리스트가 산출된다. 그늘막 설치는 저비용·단년도 집행 사업이므로 "데이터 기반 후보지 선정 → N곳 설치 → 일평균 이용객 N명 수혜(승하차 실측으로 정량화)"라는 성과 지표가 바로 성립하며, 특히 시가 추진 중인 **유개식 승강장 확대(716→980곳)의 잔여 후보지 선정**에 즉시 활용 가능하다.
- 정류장 단위 통합 테이블은 교통·안전·복지 부서가 공유 가능한 기초 자료(부서 간 데이터 칸막이 해소 사례).

**ㅇ 계절·대상 확장 로드맵**
- v1 여름(그늘·폭염) → v2 겨울(바람막이·온열의자 — 6절 요청 데이터 확보 시) → v3 야간 상시(조명·안전) → 교통약자 이동지원 정책 연계. 동일한 데이터 구조에 계절 레이어만 교체되는 설계라 확장 비용이 낮다.
- 축적되는 정류장 단위 수요·시설·불편 데이터는 향후 노선·배차 검토의 기초자료로도 활용 가능하다.

**ㅇ 확장성·이식성·파급효과**
- 정류장·노선·승하차·시설 데이터는 전국 지자체가 유사 형식으로 공개하고 있어 동일 파이프라인을 타 도시에 이식 가능. 전 코드·전처리 과정을 공개 저장소로 제공하여 공공데이터 활용 문화 확산이라는 대회 취지에 부합.
- 폭염·한파 옥외 노출 감소라는 건강 편익과 시설 투자 효율화라는 재정 편익을 동시에 발생시키고, 대기 경험 개선은 장기적으로 대중교통 이용 활성화 정책과 정합한다.

| **추가 요청 데이터** | ① 버스정류장 시설물 현황 — 승차대(쉘터)·의자·온열의자 유무(정류장번호 기준) ② 춘천시 지정 무더위쉼터 현황(명칭·주소·좌표·운영시간) ③ 폭염대비 접이식 그늘막의 위경도 좌표(현재 주소만 공개) ④ 최근 3년 온열질환자 발생 통계(읍면동 단위 가능 시) ⑤ 버스노선별 시간대별 승하차 인원의 **전체 기간 원본** — 현재 공개 파일은 65,535행(구형 엑셀 행 한계)에서 절단되어 2025.6.25~28 약 4일치만 담고 있음을 원본 검증으로 확인함 |
| **요청 데이터 활용 목적** | ①: 대기환경 카드의 '미확인'을 '확인'으로 전환 — 자체 실사 150곳을 전체 정류장으로 즉시 확장(온열의자 현황은 겨울 확장 v2의 핵심). ②: 폭염특보 시 쉼터 경유 안내. ③: 지오코딩 오차 제거. ④: 폭염 취약지 진단의 결과 검증 지표. ⑤: 계절별 수요 비교(여름/겨울 프리셋) |
| **필요한 데이터 형식** (CSV / JSON / API 등) | CSV (좌표 포함 시 WGS84 위경도) |
| **희망 제공 범위** | 춘천시 전역, 최신 기준일 스냅샷 1회 제공이면 충분 (⑤는 행 수 제한 없는 전체 기간) |
| **해당 데이터가 필요한 이유 및 기대 효과** | 본 서비스는 요청 데이터 없이도 자체 실사와 3상태 표기로 완결적으로 동작하도록 설계되어 있다. 다만 시설물 현황(①)이 제공되는 순간 서비스 커버리지가 전체 정류장으로 확장된다 — 즉 본 서비스는 춘천시의 데이터 개방이 시민 서비스 품질을 직접 향상시키는 구조를 시연으로 증명하는 사례가 되며, 정류장 시설물 데이터는 전국적으로 공개 사례가 드물어 제공 시 춘천시가 데이터 개방 선도 사례가 될 수 있다. ⑤는 공개 데이터의 품질 문제(행 절단)를 참가팀이 발견·제보하는 것 자체가 대회 취지에 부합하는 사례다. |
