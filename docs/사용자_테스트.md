# 쉼표 정류장 — 검증 계획

> 파생 문서. 제품 성공을 주장하기 위한 최소 검증 계약이며 `제품_기획.md`의 범위를 변경하지 않는다.

## 고령 사용자 과업

대상: 스마트폰 숙련도가 서로 다른 65세 이상 5명 이상. 보호자는 관찰만 하고 대신 조작하지 않는다.

1. 공통 QR로 시민 앱 열기
2. 첫 화면에서 `목적지로 가는 길 찾기` 선택
3. 목적지를 말하고 인식 결과 확인
4. 자동 산출된 승차 정류장·버스·도착시간·시설 확인
5. 결과 화면에서 “도착 안내 화면이 꺼져 있다”를 음성 또는 버튼으로 신고
6. AI 구조화 결과의 오류를 찾고 수정·제출
7. 첫 화면으로 돌아가 `정류장 불편 알리기` 직접 진입도 수행

측정: 과업 성공률, 완료시간, 도움 요청, 잘못 선택한 정류장, 음성 재시도, AI 수정 횟수. 수정 전후를 같은 과업으로 비교한다.

## 길찾기 결과 검증

- 총 소요시간이 `승차 정류장까지 도보 + 대기 + 버스 탑승 + 하차 후 도보`와 일치하는지 확인
- 가장 먼저 오는 버스와 가장 빨리 목적지에 도착하는 버스가 다른 사례 포함
- 버스 도착예정이 도보시간보다 짧은 후보가 추천에서 제외되는지 확인
- 이동 여유시간 기본값을 고령 사용자 실측으로 조정
- 한 화면에서 버스번호·총 소요시간·도착예정·도보시간·목적지 도착예상을 이해하는지 확인

## 정류장 식별 검증

- 맞은편 정류장 5쌍 이상을 경로 엔진 결과로 검증
- 정류장명이 같은 사례, 4자리 번호가 다른 사례, GPS 오차 사례 포함
- 목적지와 노선 진행방향 없이 거리만으로 승차 정류장을 정하지 않았는지 확인
- 결과 화면의 정류장명·번호·방면·랜드마크가 승차 위치 확인에 도움이 되는지 기록

## AI 검증셋

- 익명화한 발화 예시를 시설별·문제별로 균형 있게 구성
- 쉬운 표현, 방언, 복수 문제, 모호한 표현, 개인정보 포함 문장 포함
- 정답 라벨은 팀원 2명이 독립 작성하고 불일치는 합의

평가:

- 시설종류·문제유형 정밀도/재현율
- 사실 주장과 정책 요구 구분 정확도
- 복수 문제 누락률
- 개인정보 마스킹 성공률
- 잘못된 정류장 자동 확정 0건
- 근거 없는 행정 권고 발생 건수

데모 예시 성공을 전체 정확도처럼 발표하지 않고 실패 사례와 폴백을 함께 제시한다.

## 행정 사용성 검증

담당자 또는 대리 평가자가 다음 과업을 수행한다.

1. 신규 제보 원문과 AI 구조화 결과 비교
2. 공식 BIT·시설자료와 충돌 근거 확인
3. 중복 묶음 해제 또는 병합
4. 현장확인 승인·반려
5. 확인 결과 입력 후 시민 화면 반영 확인

측정: 건당 검토시간, 잘못 승인한 비율, 근거를 찾는 클릭 수, 상태변경 감사로그 이해도.

## 실패·안전 테스트

- 위치 거부, GPS 오차, 맞은편 후보 겹침
- 도착 API 지연·실패·대체정보
- STT 미지원·오인식·TTS 자기음성 재인식
- AI 서버 실패와 수동 버튼 폴백
- 중복 제출·도배·개인정보가 포함된 자유서술
- 시민 제보와 공식자료 충돌
- 오프라인 최초 접속과 재방문 차이

## 합격 기준

- 핵심 과업은 로그인 없이 완료
- AI 실패 시에도 버튼 제보와 관리자 수동 분류 가능
- 사람 확정 전 공식 시설상태 자동 변경 0건
- 미확인을 없음으로 바꾼 사례 0건
- 서비스 조회를 실제 승차로 표시한 화면·지표 0건
- 모든 AI 권고에 원문·출처·기간·불확실성·모델 버전 표시
