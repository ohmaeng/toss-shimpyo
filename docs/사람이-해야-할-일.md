# 사람이 해야 할 일

코드는 여기까지 왔다. 아래는 **AI가 대신 할 수 없는 것**들이다.
날짜는 적지 않는다 — 일정은 팀이 조절한다. 대신 **의존 순서**로 적는다.

---

## 0단계 — 승인 대기가 있는 것 (제일 먼저, 기다리는 동안 다른 걸 한다)

### 앱인토스 콘솔 — 입점 요건 확인
developers-apps-in-toss.toss.im

- [ ] **개인(비사업자) 개발자로 미니앱 출시가 가능한가?**
- [ ] **콘솔 심사에 며칠 걸리는가?** ← 이 값만큼 실질 마감이 앞당겨진다
- [ ] 출품 요건이 '제출'인가 '심사 통과 후 출시'인가?
- [ ] 콘솔이 요구하는 제출물 목록 확보

**이 둘을 모르면 남은 일정을 계산할 수 없다.** 코드는 바이브코딩으로 압축되지만 입점 절차는 압축되지 않는다.

### 공공데이터포털 — 운영계정 활용신청
개발계정은 **일 1,000건** 한도다. 승인에 지연이 있으므로 가장 먼저 신청해야 심사 기간에 쓸 수 있다.

> 프록시에 CDN 캐싱을 넣어 상류 호출을 정류장당 25초에 1건으로 합쳤지만, 그래도 MAU 1,000 목표에서는 개발계정으로 부족하다.

## 2단계 — 차별점이 실제로 존재하는지 측정 (키 받은 직후)

```bash
cd pipeline && python build_data.py --probe --city-codes 23,25,51
```

쿼터를 거의 쓰지 않고 **배차간격 확보율**만 잰다.
**20% 미만이면 대안 정류장 제안은 화면에 거의 안 뜬다** — 4중 조건 #4가 대부분 탈락하기 때문이다. 판정문이 출력된다.

이게 카카오맵과 구별되는 유일한 지점이므로, **이 숫자를 제일 먼저** 봐야 한다. 낮으면 컷 규칙이 발동하고(§7.3), 차별점을 다른 축에서 찾아야 한다.

## 4단계 — 프록시 배포 (목 모드를 끄는 스위치)


## 5단계 — 실데이터 빌드

```bash
cd pipeline
# 무더위쉼터 CSV를 공공데이터포털에서 내려받은 뒤
python build_data.py --city-codes 23,25 --shelter-csv ./shelters.csv --data-date 2026-07
```
리포트의 세 숫자를 반드시 확인하라: **TAGO 조인 성공률 · 쉼터 근접 매칭률 · 배차간격 확보율**.

> 쉼터 CSV에 운영시간 컬럼이 없으면 전국 쉼터가 "운영시간 미확인"으로 표기된다. 그게 정직한 동작이지만, 컬럼이 있는 배포본을 찾는 편이 낫다.

# 키 목록 — 무엇을 어디서 발급받아 어디에 넣는가

## ① 공공데이터포털 serviceKey (필수)

**발급처**: [data.go.kr](https://www.data.go.kr) 회원가입 → 아래 **5개 서비스 각각 활용신청** → 마이페이지에서 **인증키(일반 인증키) 1개**를 받는다.

> 키는 계정당 하나지만, **서비스마다 개별 활용신청**을 해야 그 서비스에 접근할 수 있다. 5개 전부 신청하라.

| 서비스 | 제공기관 | 코드가 쓰는 곳 |
|---|---|---|
| **버스도착정보** `ArvlInfoInqireService` | 국토교통부 TAGO | `proxy/api/arrivals.ts` — 실시간 도착정보 (코어 기능) |
| **버스정류소정보** `BusSttnInfoInqireService` | 국토교통부 TAGO | `pipeline/sources.py` — 전국 정류장 위치 |
| **버스노선정보** `BusRouteInfoInqireService` | 국토교통부 TAGO | `pipeline/sources.py` — 노선-정류장 매핑 + **배차간격** (4중 조건 #4) |
| **단기예보** `VilageFcstInfoService_2.0` | 기상청 | `proxy/api/weather.ts` — 초단기실황 기온(T1H) |
| **기상특보** `WthrWrnInfoService` | 기상청 | `proxy/api/weather.ts` — 폭염특보 |

**주의**: 포털이 *Encoding*/*Decoding* 두 형태의 키를 준다. 코드가 `URLSearchParams`로 인코딩하므로 **Decoding 키(원본)** 를 넣어라. Encoding 키를 넣으면 `%2F` 가 이중 인코딩되어 401이 난다.

**넣는 곳** (클라이언트에 절대 노출 금지):
```bash
# proxy/.env  (그리고 Vercel 대시보드의 환경변수)
DATA_GO_KR_SERVICE_KEY=여기에_디코딩된_serviceKey

# 파이프라인 실행 시에도 같은 키를 환경변수로
export DATA_GO_KR_SERVICE_KEY=...
```

## ② Kakao JavaScript 키 (지도 ① — 선택이지만 권장)

**발급처**: [developers.kakao.com](https://developers.kakao.com) → 애플리케이션 추가 → **JavaScript 키**
**추가로 필요**: 플랫폼 > Web > **사이트 도메인 등록** (토스 WebView가 노출하는 origin). 미등록이면 SDK가 로드되지 않는다.

```bash
# app/.env
VITE_KAKAO_JS_KEY=...
```

> JS 키는 클라이언트에 노출되는 것이 **정상**이다(설계상 공개 키). 대신 반드시 콘솔에서 도메인 제한을 걸어라.
> 미설정이면 지도 폴백이 ②Leaflet부터 시작한다.

## ③ VWorld 인증키 (지도 ② — 프로덕션 필수)

**발급처**: [vworld.kr](https://www.vworld.kr) → 오픈API → 인증키 발급 (활용 URL 등록 필요)

```bash
# app/.env
VITE_TILE_URL=https://api.vworld.kr/req/wmts/1.0.0/발급받은키/Base/{z}/{y}/{x}.png
VITE_TILE_ATTRIBUTION=© VWorld
```

> **왜 OSM을 못 쓰나**: `tile.openstreetmap.org`는 OSMF Tile Usage Policy상 상용·대량 트래픽 사용이 금지되어 있다. 전국 MAU 1,000+ 목표와 정면 충돌한다.
> 미설정이면 코드가 지도 ②를 **건너뛰고 리스트 UI(③)로 내려간다** — 조용히 위반하지 않는다. (개발 모드에서만 OSM을 쓴다.)

## 키가 아닌 설정값 (발급 불필요, 직접 채움)

| 변수 | 파일 | 비우면 어떻게 되나 |
|---|---|---|
| `VITE_PROXY_BASE` | `app/.env` | **목 모드로 동작** (상단에 예시 데이터 배너). 프록시 배포 후 URL을 넣으면 실데이터로 전환 |
| `VITE_DATA_BASE` | `app/.env` | 앱의 `/data` 를 쓴다(개발용). CDN에 올렸으면 그 베이스 URL |
| `VITE_REPORT_FORM_URL` | `app/.env` | 제보 버튼이 숨겨진다 (죽은 버튼을 보여주지 않는다) |
| `VITE_PRIVACY_POLICY_URL` | `app/.env` | 개인정보 처리방침 링크가 숨겨진다 (**제출 전 필수**) |
| `VITE_TILE_ATTRIBUTION` | `app/.env` | 타일 저작권 표기 없음 (타일 제공자 약관 위반 가능) |

## 발급 없이 받는 데이터

- **무더위쉼터**: 행정안전부 데이터를 [data.go.kr](https://www.data.go.kr)에서 **CSV로 직접 다운로드**한다. API 키 불필요. `--shelter-csv` 로 파이프라인에 넘긴다.

## 설정 요약

```bash
cp app/.env.example app/.env
cp proxy/.env.example proxy/.env
```

```
proxy/.env      → DATA_GO_KR_SERVICE_KEY 하나만. 절대 커밋 금지.
app/.env        → VITE_KAKAO_JS_KEY, VITE_TILE_URL, VITE_PROXY_BASE, (나머지 선택)
```

**클라이언트에 들어가도 되는 것**: Kakao JS 키, VWorld 타일 URL (둘 다 도메인 제한으로 보호)
**절대 들어가면 안 되는 것**: `DATA_GO_KR_SERVICE_KEY` — 프록시가 존재하는 유일한 이유다

확인 방법:
```bash
cd app && npx vite build && grep -rE "serviceKey|DATA_GO_KR" dist/   # 아무것도 안 나와야 한다
```

## 결정이 필요한 것 (AI가 못 정한다)

### 대안 정류장 제안을 유지할 것인가
세 페르소나가 이 기능에 대해 정반대로 반응했다:
- **직장인**: "안 본다. 내 버스만 보면 된다."
- **고령자**: "뙤약볕에 3분을 걸으라는 거다. 놓치면 30분을 더 기다린다."
- **유모차 보호자**: "직선거리 3분은 유모차로 5분이 넘는다. 그 도박은 못 한다."

**이상적 수신자가 아무도 없는 기능**이라는 지적이 나왔다. 반면 이것이 카카오맵과 구별되는 유일한 지점이기도 하다.

→ 2단계의 배차간격 확보율 숫자를 먼저 보고 결정하라. 확보율이 낮으면 이 논쟁 자체가 무의미하다.
