# 쉼표 정류장 — 데이터·AI 계약

> `제품_기획.md`의 데이터 및 AI 원칙을 구현 가능한 계약으로 구체화한다. 제품 범위를 새로 정의하지 않는다.

## 데이터 계층

| 계층 | 예시 | 확정성 | 허용 용도 |
|---|---|---|---|
| 공식 원본 | 춘천시 위치·노선·BIT, 국토부·TAGO | 출처·기준일 범위 내 공식 | 정류장·운영·설치 근거 |
| 검수 관측 | 현장조사, 담당자 확인 | 확인일 기준 확정 | 시설상태 갱신 |
| 민간 참고 | 카카오맵 표시명·지도 객체 | 교차검토 참고 | 현행성·명칭 차이 확인 |
| 시민 신호 | 음성·버튼 불편, 화면 조회 | 검수 전·비대표 표본 | 반복·충돌·조사 필요 탐지 |
| AI 산출 | 구조화·중복군·권고 | 자동 생성·미확정 | 담당자 검토 지원 |

## 핵심 테이블

- `stops`: 내부 정류장 ID, 공식 ID, 4자리 번호, 명칭, 좌표, 방면, 운영상태, 적용기간
- `source_records`: 출처기관, 원본 ID, 기준일, 원본값, 내부 정류장 연결상태
- `facility_observations`: 시설종류, 상태, 위치관계, 출처, 관측일, 검수자, 확정여부
- `citizen_reports`: 확인된 정류장, 시간구간, 원문, 구조화 유형, 동의, 처리상태
- `report_clusters`: 정류장·시설·문제별 중복군, 포함 제보, 기간, 규칙/모델 버전
- `ai_recommendations`: 과업유형, 근거목록, 불확실성, 모델 버전, 생성시각
- `survey_cases`: 담당자 결정, 현장확인 결과, 상태변경, 감사로그
- `stop_daily_aggregates`: 소수 억제된 일·주·월 집계

## 정류장 연결 판정

1. 공식 동일 ID 또는 공식 대응표: 자동 확정
2. 좌표·노선·정차순서·방면 복수 일치: 높은 신뢰 후보, 규칙 기준 충족 시 확정
3. 명칭·좌표만 유사: AI 검토 후보, 자동 확정 금지
4. 맞은편·동명·운영시점 충돌: 사람 검수 전 연결 금지

모든 연결에 `방법`, `사용 근거`, `기준일`, `검수상태`를 남긴다.

## AI 입력과 출력

### 입력

- 시민이 확인한 정류장 ID
- 음성 인식 원문 또는 선택 항목
- 공식 시설·BIT·운영 상태와 기준일
- 같은 정류장의 최근 구조화 제보

### 출력

- 시설종류와 문제유형 후보
- 사실 주장/정책 요구 구분
- 지속성·긴급 표현 후보
- 유사 제보 중복군
- 공식자료와의 일치·충돌 설명
- 행정 과업 유형과 근거가 포함된 권고
- 불확실성과 사람 확인 필요사항

사용자에게 구조화 결과를 제출 전에 보여준다. 관리자에게는 원문, 규칙 결과, AI 결과, 출처를 함께 보여준다.

## 판단 책임

| 처리 | 담당 |
|---|---|
| 공식 수치 집계·ID 직접 일치 | 규칙/SQL |
| 음성 구조화·유사 제보 묶기·충돌 설명 | AI |
| 애매한 정류장 대응 승인 | 데이터 관리자 |
| 시설상태 확정·현장조사 결과 | 담당 공무원/조사자 |
| 공사·예산·법적 민원 결정 | 권한 있는 행정 주체 |

## 평가 지표

- 구조화 정확도: 시설·문제유형 정답률
- 정류장 연결 정확도: 확정 표본 정밀도와 미결 비율
- 중복 묶음 품질: 잘못 합친 비율·놓친 중복 비율
- 과업 유효성: 현장확인 후 실제 불일치/고장 확인 비율
- 행정효율: 건당 검토시간과 데이터 갱신 소요기간
- 시민환류: 확정 결과가 시민 화면에 반영된 비율

모델 성능은 별도 검증셋으로 측정하고, 데모 예시 몇 건을 전체 정확도처럼 발표하지 않는다.

## 개인정보·보관

- 위치 원본은 정류장 후보 계산 후 폐기하고 확인된 정류장 ID만 전송한다.
- 음성 원본 저장은 선택 동의와 짧은 보관기간을 전제로 한다. 기본은 텍스트 변환 후 음성 폐기다.
- 자유서술에서 전화번호·주소 등 개인정보를 탐지·마스킹한다.
- 반복 식별용 영구 기기 ID를 만들지 않는다.
- 소수 집계를 외부에 공개하지 않는다.
- 모델·규칙·사람의 변경 이력을 감사로그에 남긴다.
