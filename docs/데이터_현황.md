### 매칭률 표기 주의 (발표 시 혼용 금지)

- **95% (899명 중 859)** = 승하차 CSV의 고유 **정류장명 단위** 매칭률 (spec §2)
- **88.2% (1667/1890)** = 마스터 **정류장 개수 단위** 수요 커버리지 (본 문서 §1)
- 분모가 다른 별개 지표. 발표·문서에는 **88.2%(정류장 단위)를 기본**으로 쓰고, 95%는 조인 방식 설명에만 사용.

## 3. 데이터 관련 정직성 규칙 (발표·화면 공통)

1. 시설은 3상태만. 미확인을 "없음"으로 쓰지 않는다.
2. 배지: "로드뷰 확인 (촬영 YYYY.MM)" 또는 "○○대장 기준". "현장 확인" 금지.
3. 수요 순위엔 항상 "양방향 합산 기준" 병기 (예: 인성병원 = 한낮 승차 4위, ID 단위로는 6위).
4. 승하차 CSV는 엑셀 65535행에서 절단(실데이터 2025.6.25~28 4일치). 농촌 정류장 223곳은 원본에 없어 수요 미확인 — 조작하지 않음.
5. 승하차는 2025.11 노선 조정 이전 자료 → 정류장 단위 수요로만 사용, 노선 단위 주장에 쓰지 않는다.

## 4. 재현 방법

```bash
cd pipeline && pip install -r requirements.txt && python build_stops.py
```
→ `data/`의 6개 CSV를 결합해 `app/public/data/stops.json` 재생성. 로드뷰 양식을 채워 `data/`(또는 지정 경로)에 두면 자동 반영.
