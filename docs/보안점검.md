## 1. API 키 노출 — 통과

- **하드코딩 없음.** `git grep` 결과 소스에 실제 TAGO 키 문자열 없음. `app/src/lib/arrivals.ts:33` 은
  `import.meta.env.VITE_TAGO_KEY` 런타임 주입 방식만 사용. serviceKey·nodeId 모두 `encodeURIComponent` 처리.
- **`.env` 격리.** 저장소 루트 `.gitignore` 에 `.env`, `app/.env` 포함. `git ls-files` 로 확인 시
  **커밋된 env 파일은 `app/.env.example` 하나뿐**(값 비어 있는 플레이스홀더 `VITE_TAGO_KEY=`).
- **번들 검사.** `app/dist/` 는 gitignore 대상이라 커밋 경로로 키가 새지 않음.

### 잔여 리스크 (문서화 필수)
- Vite 는 `VITE_` 접두 환경변수를 **빌드 시 클라이언트 번들에 인라인**한다. 따라서 로컬 `app/.env` 에
  실제 키를 넣고 빌드하면 그 키가 `dist/assets/index-*.js` 안에 평문으로 박힌다(실측 확인함).
  이는 서버가 없는 정적 앱에서 클라이언트가 직접 외부 API를 호출하는 구조상 **불가피**하다
  (브라우저에 전달되는 순간 키는 사용자에게 노출됨).
- **완화책:**
  1. 데모/배포는 **무키 빌드** 권장 — 키 없으면 `getArrival` 이 즉시 `배차간격 약 N분` 폴백으로 동작하므로
     기능 손실이 UI상 없음.
  2. 실키를 쓸 경우 공공데이터포털(data.go.kr)에서 해당 키의 **일일 호출 쿼터·트래픽 제한**에 의존해
     남용 피해를 한정한다(도메인/Referer 제한은 data.go.kr 미지원).
  3. 근본 해결은 키를 감추는 **서버 프록시**지만, "서버 없는 정적 앱" 제약상 범위 밖 → 향후 과제로 기록.

## 2. URL 파라미터 주입 / XSS — 통과 (수정 불필요)

보호자 공유 링크 `?fav=id1,id2` 처리 경로를 코드로 검증:

- `app/src/features/share/shareLink.ts` `parseShareParam(search, validIds)` 은
  **로드된 stop.id 화이트리스트와의 교집합만** 통과시킨다(`if (!whitelist.has(id)) continue`).
  악성 입력(`<script>`, 없는 id, 특수문자)은 화이트리스트에 없으므로 전량 폐기.
  잘못된 `decodeURIComponent` 는 `try/catch` 로 폐기, 중복은 `Set` 으로 제거.
- 소비처 `app/src/features/share/ImportOnLoad.tsx:25-26` 은 `validIds = stops.map(s => s.id)`
  (실제 로드된 정류장 id)를 넘긴다 → 화이트리스트가 신뢰 데이터로 고정됨.
- 통과한 id 는 `useFavorites.addMany` 로 **즐겨찾기 id 배열에만 저장**되고, 렌더 시에는 id 로 정류장을
  역참조해 표시(원문 문자열을 DOM에 직접 삽입하지 않음). React 기본 이스케이프까지 더해 XSS 경로 없음.
- 회귀 방지 테스트 존재: `shareLink.test.ts` 가 `<script>`·미존재 id·중복·라운드트립을 커버.

## 4. CSP — 수정 (메타 추가 + 실측 검증)

`app/index.html` `<head>` 에 다음 CSP 메타 태그를 추가:

```
default-src 'self';
img-src 'self' data: https://*.tile.openstreetmap.org;
connect-src 'self' https://*.tile.openstreetmap.org http://apis.data.go.kr https://apis.data.go.kr;
style-src 'self' 'unsafe-inline';
script-src 'self';
worker-src 'self';
object-src 'none';
base-uri 'self'
```

### 정책 근거 (지시별)
- `default-src 'self'` — 기본은 동일 출처만 허용(화이트리스트 방식).
- `img-src 'self' data: https://*.tile.openstreetmap.org` — Leaflet 마커는 data URI, 지도 타일은 OSM.
- `connect-src ... OSM ... apis.data.go.kr` — 타일 런타임 캐시 fetch + TAGO 도착정보 API.
  코드는 `https://apis.data.go.kr` 만 호출하지만, 일부 data.go.kr 엔드포인트의 http 리다이렉트 대비해
  `http://apis.data.go.kr` 도 함께 허용(과잉 차단으로 데모가 깨지지 않도록).
- `style-src 'self' 'unsafe-inline'` — Leaflet 이 요소 `style` 속성(transform 등)과 인라인 스타일을 사용.
  스타일 한정이며 스크립트는 `'unsafe-inline'` 을 부여하지 않음.
- `script-src 'self'` — **외부 스크립트 전면 차단.** vite-plugin-pwa 의 SW 등록 스크립트가
  인라인이 아닌 **외부 `registerSW.js`(동일 출처)** 로 생성됨을 dist 검사로 확인 → 인라인 예외 불필요.
- `worker-src 'self'` — PWA 서비스워커(`sw.js`).
- `object-src 'none'` / `base-uri 'self'` — 플러그인·`<base>` 하이재킹 하드닝(앱 미사용, 부작용 없음).

### 실측 검증 (`npm run build && npm run preview`)
- 헤드리스 브라우저로 preview 로드 → **콘솔 CSP 위반/에러 0건**.
- Leaflet 지도 정상 렌더: `.leaflet-container` 존재, 타일 **24/24** 이미지 로드(naturalWidth>0),
  OSM 타일 요청 `200 OK`(a/b/c 서브도메인).
- 정류장 카드("배차간격 약 15분" 폴백), "가족에게 공유"·"안내문 인쇄" 버튼 등 UI 정상.
- 결론: CSP 가 실제 앱을 깨뜨리지 않음.

> 참고: `frame-ancestors` 는 `<meta>` 방식에서 무시되므로 미포함. 클릭재킹 완전 차단이 필요하면
> 배포 서버 응답 헤더로 `Content-Security-Policy: frame-ancestors 'none'` 또는 `X-Frame-Options: DENY`
> 를 추가할 것(정적 호스팅 헤더 설정 영역, 코드 범위 밖).
