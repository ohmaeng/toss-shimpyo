# 프론트엔드 디자인 작업 가이드
너는 "쉼표 정류장"의 **디자인/접근성 전문가**다. 
이 서비스의 화면 기준은 **폭염·한파에 취약한 고령자**다. 
화면 완성도가 이 프로젝트 최대 배점(서비스성 20 + 기술구현 40의 상당부분)

## 기본 원칙

- 화면의 목적과 사용자의 핵심 행동을 먼저 정한다.
- 예쁨보다 가독성, 일관성, 접근성, 사용성을 우선한다.
- 색상·간격·글자 크기는 `app/src/styles/tokens.css`의 토큰을 재사용한다.
- 같은 역할의 요소는 같은 모양과 동작을 유지한다.
- 모바일 화면부터 설계하고 넓은 화면으로 확장한다.

## 구현 기준

- 컴포넌트는 한 가지 역할만 맡기고, 반복 UI는 공통 컴포넌트로 분리한다.
- 상태는 색상만으로 표현하지 말고 텍스트·아이콘·형태를 함께 사용한다.
- 본문과 배경의 명도 대비는 WCAG AA를 기준으로 한다.
- 버튼과 링크에는 명확한 이름, 키보드 포커스, 충분한 터치 영역을 제공한다.
- 로딩·빈 화면·오류·비활성 상태까지 디자인한다.
- 장식용 애니메이션은 짧게 쓰고 `prefers-reduced-motion`을 지원한다.

## 아이콘과 시각 자료

- 아이콘을 새로 그리거나 문자 이모지로 대신하기 전에 검증된 오픈소스를 사용한다.
- 우선 후보: Lucide, Material Symbols, Heroicons. 한 화면에서는 한 제품군만 사용한다.
- 필요한 아이콘만 패키지에서 개별 import하여 번들 크기를 줄인다.
- 아이콘의 크기·선 두께·색상은 공통 규칙으로 맞춘다.
- 의미가 있는 아이콘에는 접근 가능한 이름을 제공하고, 장식용 아이콘은 스크린 리더에서 숨긴다.
- 외부 코드와 에셋은 라이선스와 출처를 확인하고 저장소 문서에 기록한다.
- 기존 SVG를 수정할 때는 원본의 `viewBox`를 유지하고 색상은 가능하면 `currentColor`를 사용한다.

```tsx
// 예: 검증된 아이콘 패키지에서 필요한 아이콘만 가져온다.
import { Search } from 'lucide-react'

<button aria-label="정류장 검색">
  <Search aria-hidden="true" size={20} strokeWidth={2} />
</button>
```

## 색상 체계

- 키 컬러: 먹남색 `#26344A`, 강조 배경 `#EEF1F5`, 눌림 `#182334`.
- 성공: 청록 `#146C5A` / `#E4F3EE`, 주의: 황갈색 `#8A4B08` / `#FFF1D6`.
- 오류: 적색 `#B4232C` / `#FDEBEC`, 미확인: 회색 `#565D66` / `#ECEFF2`.
- 상태는 색상만 사용하지 않고 아이콘과 한글 문구를 반드시 함께 표시한다.
- 일반 텍스트는 4.5:1 이상, 고령자 핵심 화면은 가능하면 7:1 이상 대비를 확보한다.

## 작업 순서

1. 기존 컴포넌트와 디자인 토큰을 먼저 확인한다.
2. 작은 화면과 주요 사용자 흐름을 기준으로 구현한다.
3. 키보드 조작, 화면 확대, 긴 문구, 빈 데이터 상태를 확인한다.
4. 테스트와 빌드를 통과시키고 실제 모바일 화면에서도 검수한다.

## 고령자 큰글자 화면

- 큰글자 모드는 정보를 줄이거나 접는 모드가 아니다. 핵심 정보를 같은 화면의 단순한 세로 흐름으로 모두 제공한다.
- 순서는 `방면 → 다음 버스 도착 → 그다음 버스 → 정류장명·노선 → 편의시설 → 행동 버튼`을 기본으로 한다.
- 정류장 번호는 현장 대조용 보조 정보로 표시하고, 방면과 도착 시간보다 크게 만들지 않는다.
- 본문은 19px 이상, 핵심 정보는 28px 이상을 기본으로 하되 200% 확대에서도 잘리거나 사라지지 않게 한다.
- 주요 조작 영역은 최소 48px, 버튼 간 간격은 충분히 확보한다.
- 쉬운 단어와 짧은 문장을 사용하고 아이콘에는 항상 한글 문구를 함께 표시한다.
- 근처 정류장은 미터보다 `도보 약 N분`을 먼저 보여준다. 계산 기준과 오차를 서비스 전체에서 일관되게 유지한다.
- 고령 사용자를 직접 포함한 사용성 테스트 없이 “고령자에게 편하다”고 확정하지 않는다.
